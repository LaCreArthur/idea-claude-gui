# Contributing

## Language

All code, comments, PR descriptions, and documentation must be in English.

## Version Numbering

For version `v0.X.Y`, intermediate releases use: `v0.X.Y-beta1`, `v0.X.Y-beta2`, ... until final `v0.X.Y`.

## Branch Strategy

- `main` - primary branch
- `develop` - development branch

All PRs must target `develop` first. PRs with medium or high-risk issues flagged by AI review must be fixed before merging.

## Issue Labels

**Status:**
- In Progress
- Todo
- Deferred
- Done

**Priority:**
- P0 - Blocker
- P1 - High
- P2 - Medium
- P3 - Low

**Type:**
- `feat` - New feature
- `enhancement` - Feature improvement
- `bugfix` - Bug fix
- `documentation` - Docs
- `tech-debt` - Technical debt
- `testing` - Tests
